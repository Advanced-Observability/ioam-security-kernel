# Plots

This directory contains the various plots.

## Encapsulating node

First, we measure the imapct depending on the AEAD scheme with the maximum PTO data (i.e., 976 bytes).
The plot is [encap_976B.pdf](./encap_976B.pdf) generated by [plot_encap_976B.py](./plot_encap_976B.py).

Second, we measure the impact depending on the size of PTO data, ranging from 32 bytes (minimum) to 976 bytes (maximum).
The plot is [encap_pto_size.pdf](./encap_pto_size.pdf) generated by [plot_encap_pto_size.py](./plot_encap_pto_size.py).

## Transit node

First, we measure the impact of the number of IOAM data fields being protected, ranging from 4 bytes (minimum) to 48 bytes (maximum in current Linux implementation), using AES-GCM.
The plot is [transit_data_fields.pdf](./transit_data_fields.pdf) generated by [plot_transit_data_fields.py](./plot_transit_data_fields.py).

Second, we measure the impact of the chosen AEAD scheme to protect the maximum number of IOAM data fields (48 bytes in current Linux implementation).
The plot is [transit_48B.pdf](./transit_48B.pdf) generated by [plot_transit_48B.py](./plot_transit_48B.py).

## Flame graph

[flame.svg](./flame.svg) is the flame graph generated by the tool, while [flame.png](./flame.png) is the annotated one to highlight the `ioam6_gather_data` function.
